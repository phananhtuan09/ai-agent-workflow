Workflow Design Sandbox HTML

Tạo một file HTML local để reviewer cấu hình, preview và approve một workflow slice trước khi AI sinh detailed spec.

Feature: workflow-design-sandbox-html Revision: 2 Trạng thái: Chờ human review

Bối cảnh cần chốt

Đây là slice thử cho workflow mới, chủ đích đủ nhỏ để implement bằng một file HTML nhưng đủ lớn để kiểm tra design-spec, create-spec, review-spec và execute-spec.

Hiện trạng đã xác nhận

  • docs/ai/workflows/feature-standard.json đã có chuỗi design-spec, create-spec, review-spec và execute-spec.

Outcome mong muốn

  • Người review mở một file HTML, chọn workflow preset, chỉnh tối thiểu vài input, xem preview JSON và approve/export cấu hình.

Scope của slice

Trong scope

  • Một trang HTML tự chạy, có form cấu hình feature slug, mục tiêu, workflow range, preview JSON, validation, copy/download và trạng thái lỗi hoặc thành công.

Ngoài scope

  • Không sửa runtime orchestrator, schema workflow thật, backend, database, auth hoặc tích hợp multi-user.

Primary flow đề xuất

  1. Chọn preset
    Reviewer chọn range mặc định là design-spec đến execute-spec hoặc chỉnh sang một slice ngắn hơn.
  2. Nhập intent
    Trang nhận feature slug, mục tiêu, độ phức tạp mong muốn và constraint ngắn.
  3. Validate và preview
    UI báo lỗi thiếu slug, slug sai kebab-case hoặc thiếu mục tiêu trước khi cho export.
  4. Export artifact
    Reviewer copy hoặc download JSON để dùng làm input thử cho các bước workflow tiếp theo.

Quyết định cần human chốt

Mỗi lựa chọn bên dưới thay đổi behavior hoặc phạm vi. Recommendation được đánh dấu trực tiếp cùng tradeoff.

D-001
HTML sandbox có nên lưu nháp cấu hình trong trình duyệt không?

Lựa chọn này quyết định hành vi khi reviewer reload trang hoặc mở lại file local trong cùng trình duyệt.

D-002
Export chính của sandbox nên ưu tiên hình thức nào?

Sandbox chạy offline nên cách xuất dữ liệu phải đủ rõ cho người test mà không cần quyền ghi trực tiếp vào repo.

D-003
Validation nên chặn export ở mức nào?

Lựa chọn này quyết định trang là một sandbox tự do hay một artifact giúp kiểm thử workflow có kỷ luật.

Rủi ro và bằng chứng kỹ thuật

Rủi ro cần biết

  • HTML sandbox có thể tạo cảm giác như workflow thật đã được tích hợp; UI cần gọi rõ đây là artifact thử nghiệm offline.

Codebase evidence

  • docs/ai/workflows/feature-standard.json - workflow pilot đã định nghĩa các bước cần test từ design-spec đến execute-spec.

Chốt hoặc yêu cầu chỉnh sửa

Approve sẽ ghi decision manifest cho step create-spec. Request changes sẽ lưu feedback để agent sửa HTML và tăng revision.
