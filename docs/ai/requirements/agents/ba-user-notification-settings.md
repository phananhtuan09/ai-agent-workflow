# Business Analysis: User Notification Settings

> Generated by BA Agent | Date: 2026-05-01

---

## 1. Problem Statement

### Current Situation
- Users receive email notifications with no control over frequency or timing
- No interface exists for users to opt out of email notifications
- No quiet hours / do-not-disturb mechanism exists

### Desired Outcome
- Users can navigate to Settings > Notifications and toggle email notifications on/off
- Users can set a quiet hours time range; no emails are sent during that window
- Settings are persisted per user and timezone-aware

---

## 2. Users & Stakeholders

### Primary Users

| User Type | Description | Goals |
|-----------|-------------|-------|
| Authenticated App User | Logged-in user with an existing profile | Control email notification delivery and quiet hours |

### Stakeholders
- Product team: Reduce notification-related complaints and unsubscribes

## 3. User Stories

### Must-Have (P0)

| ID | As a... | I want to... | So that... |
|----|---------|--------------|------------|
| US-01 | authenticated user | toggle email notifications on or off | I stop receiving emails when I don't want them |
| US-02 | authenticated user | set a quiet hours start and end time | emails are not sent during my sleep or focus hours |
| US-03 | authenticated user | have my quiet hours shown in my local timezone | I set the correct times without converting manually |
| US-04 | authenticated user | have my settings saved automatically after I change them | I don't lose changes when I navigate away |

### Should-Have (P1)

| ID | As a... | I want to... | So that... |
|----|---------|--------------|------------|
| US-05 | authenticated user | see the current state of my notification settings when I open the page | I know what is already configured |
| US-06 | authenticated user | receive an error message if quiet hours are invalid | I can correct mistakes before saving |

## 4. Functional Requirements

| ID | Requirement | Priority | Acceptance Criteria |
|----|-------------|----------|---------------------|
| FR-01 | Display Notifications settings page via Settings > Notifications | Must | Page renders for authenticated users; unauthenticated redirected to login |
| FR-02 | Show email notifications toggle pre-populated with current value | Must | Toggle reflects persisted state on page load |
| FR-03 | Toggle email on/off persists to user profile | Must | After toggle, refreshing page shows updated state |
| FR-04 | Show quiet hours time pickers pre-populated with stored values | Must | Fields show current saved values; empty if not set |
| FR-05 | Save quiet hours in UTC, display in user's local timezone | Must | DB stores UTC; UI displays local time |
| FR-06 | Validate start time ≠ end time | Must | Equal times show validation error, not persisted |
| FR-07 | Validate both quiet hours fields set together or not at all | Must | Partial save shows validation error |
| FR-08 | Suppress emails when notifications disabled regardless of quiet hours | Must | Disabled user receives no emails |
| FR-09 | Suppress emails during active quiet hours window | Must | Emails triggered during window are not delivered |
| FR-10 | Show success confirmation after save | Should | Toast or inline message appears after successful save |
| FR-11 | Show error message if save fails | Should | Network/server error shows user-readable message |

### Data Requirements

| ID | Requirement | Details |
|----|-------------|---------|
| DR-01 | `email_notifications_enabled` boolean | User profile table; default: `true` |
| DR-02 | `quiet_hours_start` time | User profile table; stored UTC; nullable |
| DR-03 | `quiet_hours_end` time | User profile table; stored UTC; nullable |

---

## 5. Business Rules

| ID | Rule | Condition | Action |
|----|------|-----------|--------|
| BR-01 | Email suppression — opt-out | `email_notifications_enabled = false` | Do not send any email to that user |
| BR-02 | Email suppression — quiet hours | Current UTC time within `[quiet_hours_start, quiet_hours_end)` | Do not send email |
| BR-03 | Quiet hours UTC storage | User saves quiet hours | Convert from local timezone to UTC before persisting |
| BR-04 | Quiet hours display conversion | Loading settings page | Convert stored UTC to user's local timezone |
| BR-05 | Quiet hours paired fields | Only one field provided | Reject with validation error |
| BR-06 | Quiet hours non-equal | Start time equals end time | Reject with validation error |
| BR-07 | Quiet hours clear | User clears both fields | Persist both as null |

## Handoff Summary

- Problem: Users have no way to control notification preferences; all email notifications are on by default with no quiet hours support
- Users: Authenticated app users
- Storage: Existing user profile table (3 new columns: `email_notifications_enabled`, `quiet_hours_start`, `quiet_hours_end`)
- Key FRs: Toggle email on/off, set timezone-aware quiet hours, persist settings
- Business Rules: UTC storage, local display, email-only, paired field validation, non-equal time validation
- Out of scope: Push, in-app, SMS, per-type granularity
- Open questions: Cross-midnight quiet hours support (Q-01) — not blocking
- Feature type: Full-stack (UI + API + DB migration)
- Complexity: Low-Medium
