# Business Analysis: AI Workflow Website

> Generated by BA Agent | Date: 2026-03-28

---

## 1. Executive Summary

A static Next.js 14 marketing and documentation website for the AI Agent Workflow open-source project, hosted on Vercel and living inside the existing repo under `website/`. The site targets "vibe coders" — developers who want to quickly adopt AI workflow tooling — and provides a landing page, guided install flow, animated workflow visualizer, and an explorable skills/commands reference. All data is sourced from static TypeScript config files with no backend or runtime API calls.

---

## 2. Problem Statement

### Current Situation

The AI Agent Workflow project lives on GitHub with documentation spread across markdown files and a CLI tool. There is no dedicated web presence to introduce the project, guide new users through installation, or let developers browse available skills and commands interactively.

### Pain Points

- Developers discovering the project have no polished entry point that communicates its value quickly (SaaS-style).
- The install process requires reading GitHub README markdown manually with no copy-paste affordance.
- Workflow logic (how agents connect and sequence) is invisible — there is no visual representation.
- Skills and commands are not browsable or filterable by AI tool without reading raw files.

### Desired Outcome

A publicly accessible website (`*.vercel.app`) that:
- Convinces a vibe coder to adopt the tool within 60 seconds on the landing page.
- Lets any developer copy the correct install command in two clicks.
- Makes the workflow architecture visually legible through animated graphs.
- Provides a full, filterable skills/commands reference with individual detail pages.

## 5. Functional Requirements

| ID | Requirement | Priority | Acceptance Criteria |
|----|-------------|----------|---------------------|
| FR-01 | Home page: product landing page with headline, feature highlights, and CTA to Install page | Must | Page loads at `/`; CTA visible above the fold; clicking navigates to `/install` |
| FR-02 | Install page: list of AI tools; selecting one updates the displayed curl command | Must | Each tool option is clickable; command text changes; matches tool-specific install string |
| FR-03 | Install page: one-click copy button for the curl command | Must | Clicking copies to clipboard; visual confirmation appears and resets after 2–3s |
| FR-04 | Install page: default/placeholder state when no tool is selected | Must | Either a default tool is pre-selected or a clear prompt is shown |
| FR-05 | Workflow Visualizer: animated graph with sequential node/edge appearance (Framer Motion) | Must | Graph animates on page load; nodes appear before connecting edges |
| FR-06 | Workflow Visualizer: click node → tooltip with label and description | Must | Clicking shows tooltip; dismissible by clicking elsewhere |
| FR-07 | Workflow Visualizer: multiple graphs from TypeScript config objects | Should | At least 2 graphs render; adding a config object adds a new graph |
| FR-08 | Skills Explorer: list/grid of all skills from TypeScript config | Must | All config entries appear on `/skills` |
| FR-09 | Skills Explorer: single-select filter by AI tool | Must | Filter controls visible; selecting updates list immediately (client-side) |
| FR-10 | Skills Explorer: empty state when no skills match filter | Must | Empty state text shown when filter yields zero results |
| FR-11 | Skill detail page at `/skills/[id]` | Must | Navigating renders detail page; 404 for unknown IDs |
| FR-12 | Skill detail: trigger keywords, use cases, markdown preview, copy install command | Must | All four sections present; markdown rendered; copy button works |
| FR-13 | Language switcher: EN/VI toggle in global nav | Should | Switching changes all UI labels and static text; persisted across pages |
| FR-14 | Dark theme with gradient accents and terminal-style elements | Must | Dark background; monospace for code blocks; at least one gradient accent on Home |
| FR-15 | Fully static site — no runtime API calls or backend | Must | `next build` produces exportable static site; no runtime `fetch` calls |

## 6. Business Rules

| ID | Rule | Condition | Action |
|----|------|-----------|--------|
| BR-01 | Tool-specific install command | User selects AI tool on Install page | Display only the command mapped to that tool |
| BR-02 | Skill detail 404 | `/skills/[id]` accessed with unknown id | Render Next.js 404 page |
| BR-03 | Filter persistence | User navigates back to Skills Explorer | Previously selected tool filter restored via URL query param |
| BR-04 | Animation plays once on mount | Workflow Visualizer page loads | Step-by-step animation plays automatically; does not loop unless replay triggered |
| BR-05 | Language default | First visit, no preference stored | Default language: English |
| BR-06 | Language persistence | User switches language | Selection persisted to localStorage across pages and refreshes |
| BR-07 | Static data only | Site builds or renders | All data from local TypeScript config; no HTTP calls to GitHub API |
